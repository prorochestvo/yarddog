# Task Breakdown

## Overview

Reshape the query daemon's list endpoints toward a simpler, saner default and
drop the `archive` opt-in. Two behaviour changes, no new pagination machinery
yet (keyset/cursor is deferred — "потом подумаем как лучше"):

1. `/api/v1/metrics` and `/api/v1/pings` default to the **last 7 days** when no
   `since` is supplied (an explicit `since` still overrides). Since the hot
   window is 30 days, the default view is always served from the hot tables.
2. The `archive=true` query flag / `IncludeArchive` filter field is **removed
   end to end**. `ListMetrics`/`ListPings` become hot-only always; `spanQuery`
   loses its only two callers and is deleted. `RunByID` and `ListChecksByRun`
   keep their transparent hot+archive spanning — those are single-run lookups,
   not history browsing, and remain correct.
3. `/api/v1/runs` keeps its shape (newest-N by `limit`, no `since`); only its
   default limit rises 50 → 100.

Deep-history access (paging past the hot window into the archive) is out of
scope for this task and returns in a later cursor-pagination pass.

## Assumptions

- 7 days is a hardcoded `defaultWindow` const in `services`, not an env key
  (no config surface added; revisit if an operator ever needs to tune it).
- The default-window policy lives in `services.QueryService` (same layer that
  already clamps limits), fed by an injected `services.Clock` — not in the
  HTTP handler, keeping `gateway/httpapi` pure marshalling and testable on a
  fake clock.
- An explicit `since` older than the hot floor now returns only what hot holds
  (no archive fallback); acceptable under the scoped "7 days or latest" model.

## Tasks

### Task 1: Inject Clock + default window into QueryService
- Description: `NewQueryService(repo, clock)`. Add `const defaultWindow = 7 * 24 * time.Hour`.
  In `Metrics`/`Pings`, after `clampLimit`, apply `if f.Since.IsZero() { f.Since = clock.Now().Add(-defaultWindow) }`.
  Bump `defaultRunsLimit` 50 → 100. Drop `IncludeArchive` from `MetricsFilter`/`PingFilter` and their doc comments.
- Acceptance Criteria: no-`since` Metrics/Pings forward `Since = now-7d`; explicit `since` passes untouched; `MetricsFilter`/`PingFilter` no longer carry `IncludeArchive`.
- Pitfalls & edge cases: apply the default only on `Since.IsZero()`, never overwrite a caller value; the clock must be the daemon's real `SystemClock` in production wiring.
- Complexity: Easy

### Task 2: Strip archive from the HTTP adapter
- Description: remove the `archive` bool parsing from `parseMetricsFilter`/`parsePingsFilter` and their doc comments in `gateway/httpapi/handlers.go`.
- Acceptance Criteria: `?archive=` is ignored (no field to set); no compile refs to `IncludeArchive` remain in the package.
- Pitfalls & edge cases: leave `include_empty`/`include_unreachable` parsing intact.
- Complexity: Easy

### Task 3: Hot-only ListMetrics/ListPings, delete spanQuery
- Description: in `infrastructure/store.go`, replace the `spanQuery(...)` call in `ListMetrics`/`ListPings` with an inline hot SELECT (`... WHERE <conds> ORDER BY id DESC LIMIT ?`), drop the per-leg args loop, delete `spanQuery`. Update the two methods' doc comments.
- Acceptance Criteria: both methods query only the hot table; `spanQuery` is gone; `RunByID`/`ListChecksByRun` untouched.
- Pitfalls & edge cases: keep the sargable `id >= ? AND ts >= ?` lower-bound pair on `since` (still valid for a hot-only scan); bind args once, then the limit.
- Complexity: Easy

### Task 4: Wire the clock in cmd/yarddogd
- Description: pass the daemon's `SystemClock` into `services.NewQueryService` at `cmd/yarddogd/main.go`.
- Acceptance Criteria: daemon builds; QueryService gets a real clock.
- Complexity: Easy

### Task 5: Tests
- Description: update `services/query_test.go` (constructor arg + a new default-window subtest under `TestQueryService_Metrics`/`_Pings`, drop the IncludeArchive-forwarding subtests, fix the runs-default expectation to 100), `gateway/httpapi/handlers_test.go` (drop `?archive` and IncludeArchive assertions; adjust any full-server test that asserted a zero forwarded `Since`), `gateway/httpapi/fakes_test.go` (`newTestServer` constructor arg), and `infrastructure/store_test.go` (delete the IncludeArchive/spanQuery subtests; keep hot-only coverage).
- Acceptance Criteria: `go vet ./... && go test ./...` green.
- Pitfalls & edge cases: reuse the existing `fakeClock`; the full-server handler tests now see `Since = now-7d` on no-param requests — assert a non-zero window, not zero.
- Complexity: Medium

### Task 6: Docs
- Description: README API table + prose — drop `&archive=` from the metrics/pings rows and the `archive=true` paragraph; state the last-7-days default and the new runs default limit (100).
- Acceptance Criteria: README matches behaviour; no stale `archive` mention for the list endpoints.
- Complexity: Easy

## Execution Order
1 → 2 → 3 → 4 → 5 → 6 (5 after the production changes compile; 6 last).

## Risks
- Wire-shape/contract change: any client passing `archive=true` silently loses archive spanning. Acceptable — LAN-only, operator-owned, and archive browsing returns via cursor pagination later.
- Full-server handler tests implicitly assumed a zero default `Since`; missing one leaves a red tree (caught by the gate).

## Trade-offs
- Hardcoded 7-day window over an env key: less config surface now, at the cost of a future edit if it must be tunable. Chosen for simplicity per the scoped request.
- Deleting `spanQuery` rather than leaving it dormant: honours "no speculative structure"; it is cheap to reconstruct when cursor pagination reintroduces archive spanning.
